# Novel Eval 产品定义

## 一句话价值主张

给中文网文作者和编辑的**本地 AI 改稿评估器**——30 分钟出一份**可追溯、可执行、可对比**的五维报告。

## 目标用户

- 自评改稿的网文作者
- 小工作室编辑（本地跑，稿不外传）

## 非目标用户

- 需要 Web 上传、多租户 SaaS 的制片公司（参见燃点商业产品）
- 仅需一次性 ChatGPT 对话、不需要复现与对比的用户

## 与 ChatGPT / 燃点的差异

| 能力 | ChatGPT 对话 | 燃点平台 | Novel Eval |
|------|-------------|----------|------------|
| 评分可复现 | 否 | 是 | 是（固定 rubric + prompt 版本） |
| 原文证据可核对 | 含糊 | 是 | 是（Map excerpts + 报告高亮） |
| 改稿前后对比 | 否 | 部分 | 是（`compare` 子命令） |
| 成本可控 | 否 | 商业定价 | 是（确认屏 + 费用明细） |
| 未发表稿隐私 | 上传云端 | 上传云端 | 本地运行、离线报告 |
| 市场对标数据 | — | 搜索增强 | LLM 推断 + 免责声明（v2 接搜索） |

## 产品形态

- **入口：** CLI（`novel-eval evaluate` / `compare`）
- **交付物：** 自包含 HTML 报告 + `result.json`
- **分发：** npm / GitHub Release

## 验收标准

### 人工（决定性）

1. 1 本真实长篇（≥10 万字）跑通全流程
2. 读报告后能**据此改至少 2 章**
3. 改稿重跑后 `compare` 显示合理的维度变化方向

### 机械

1. 五维 analysis 中 ≥80% 的 `[chapterId#index]` 指针可打开证据面板
2. `compare` 对两个 result.json 输出五维 delta 与总分变化
3. 评估前确认屏章节数准确，费用预估误差 ±50% 以内

## 反 demo 原则

新功能先问：**帮用户改稿，还是只让报告好看？** 后者排后。
