## 一、项目概述

**项目名：** Novel Eval
**仓库：** `reed-soul/novel-eval`

**定位：** 个人使用的 AI 小说评估工具。上传小说文件 → 五维结构化评估 → 交互式 HTML 报告。

**核心流程：**
```
上传文件(txt/epub/pdf/docx)
  → 文档解析(统一接口层)
  → LLM 智能分章
  → Map阶段: 逐章评估(Claude Code -p)
  → Reduce阶段: 基于章节摘要做全局分析(Claude Code mcp serve)
  → 生成结构化评分数据(JSON)
  → 渲染交互式 HTML 报告(雷达图/人物关系图/情绪曲线/章节趋势)
```

**技术栈：**
- 运行环境：Node.js，跑在本地 Mac mini
- 评估引擎：Claude Code（主力，`-p` 单次调用 + `mcp serve` 批量调用）
- 抽象层：统一 AI Agent 接口，保留切换到 Cursor Agent 的能力
- 文档解析：epub / pdf-parse / mammoth / chardet + iconv-lite
- 前端：本地 Web Dashboard
- 输出：JSON 评分数据 + 交互式 HTML 报告（ECharts 图表）
- 评估规则：项目内 YAML 配置文件
- 语言：TypeScript

## 二、五维评估模型

| 维度 | 子项 | 评分范围 | 权重 |
|------|------|---------|------|
| **故事架构** | 情节完整性、叙事结构合理性、起承转合、节奏把控、悬念设置 | 0-100 | 25% |
| **人物塑造** | 角色立体感、人物弧光完整性、动机合理性、角色关系丰富度 | 0-100 | 25% |
| **文笔质量** | 语言表现力、对话自然度、修辞运用、描写细腻度、风格统一性 | 0-100 | 15% |
| **情感共鸣** | 情感张力、读者代入感、情绪节奏、共鸣深度 | 0-100 | 20% |
| **市场潜力** | 类型定位清晰度、目标受众匹配、创新性、同类作品差异化 | 0-100 | 15% |

**评估产出结构（JSON）：**
```json
{
  "novel": { "title": "", "author": "", "totalChapters": 0, "wordCount": 0 },
  "overall": { "totalScore": 0, "grade": "" },
  "dimensions": {
    "storyStructure": { "score": 0, "subscores": {}, "analysis": "", "quotes": [] },
    "characterization": { "score": 0, "subscores": {}, "analysis": "", "quotes": [] },
    "writingQuality": { "score": 0, "subscores": {}, "analysis": "", "quotes": [] },
    "emotionalResonance": { "score": 0, "subscores": {}, "analysis": "", "quotes": [] },
    "marketPotential": { "score": 0, "subscores": {}, "analysis": "", "quotes": [] }
  },
  "chapters": [
    {
      "id": 0,
      "title": "",
      "wordCount": 0,
      "scores": {},
      "summary": "",
      "keyEvents": [],
      "characters": [],
      "emotionalTension": 0
    }
  ],
  "characters": [
    {
      "name": "",
      "role": "",
      "arc": "",
      "relationships": [],
      "firstAppearance": 0,
      "keyChapters": []
    }
  ],
  "emotionalCurve": [
    { "chapter": 0, "tension": 0, "valence": 0 }
  ],
  "suggestions": [
    { "dimension": "", "type": "", "content": "", "relatedChapters": [], "quote": "" }
  ]
}
```

**核心原则：**
- 每个评分必须附带 `quotes`（原文引用）和 `analysis`（分析依据）
- 不做空泛评价，不写"值得点赞"等 AI 味评语
- 权重可在 YAML 配置中调整

## 三、评估流程（Map-Reduce 架构）

### 阶段一：解析 + 分章

```
文件上传 → 文档解析(统一接口) → 提取纯文本 → LLM 分章
  输出：[{ chapterId, title, content, startIndex, endIndex }]
```

分章用 Claude Code 单次 `-p` 调用。输入小说前 3000 字 + 后 3000 字样本（首尾都有助于识别结构），输出 JSON 格式的章节列表。对于百万字小说，分多次采样（每 10 万字采一段样本），最后合并。

### 阶段二：Map — 逐章评估

```
章节列表 → 并发调用 Claude Code -p（默认 3 并发）
  每章输入：章节正文 + 评估规则 YAML + 上下文（前一章摘要）
  每章输出：{
    scores: { 五维子项评分 },
    summary: 章节摘要,
    keyEvents: 关键事件,
    characters: 本章出场角色,
    emotionalTension: 本章情绪张力值(0-100)
  }
```

并发控制：默认 3 并发，避免 API 限流。每章之间传递前一章摘要作为上下文衔接。

### 阶段三：Reduce — 全局分析

```
所有章节评估结果 → Claude Code mcp serve（常驻）
  输入：全部章节摘要 + 人物出场记录 + 情绪张力序列
  输出：{
    characterRelationships: 人物关系图谱数据,
    emotionalCurve: 整体情绪节奏曲线,
    overallAnalysis: 各维度全局分析,
    suggestions: 跨章节改进建议（附原文引用）
  }
```

Reduce 阶段用 MCP Server 常驻，需要多次交互：先分析人物关系，再分析节奏，再生成建议，同一个会话内完成。

### 错误处理

- 单章评估失败 → 重试 1 次 → 仍失败则标记跳过，最终报告标注缺失章节
- Reduce 阶段中断 → 可基于已有章节摘要恢复，不需要重新 Map
- Claude Code 进程超时（>5min）→ kill + 重试

## 四、统一抽象层

### 核心接口

```typescript
interface AIAgentEvent {
  type: 'assistant' | 'tool_use' | 'result' | 'error';
  content: string;
  data?: any;
  usage?: { inputTokens: number; outputTokens: number; costUsd: number };
}

interface AIAgentAdapter {
  readonly name: string;

  // 单次调用（-p 模式）
  run(prompt: string, options: RunOptions): AsyncIterable<AIAgentEvent>;

  // 会话模式（mcp serve / 多轮）
  createSession(options?: SessionOptions): AIAgentSession;

  // 查询可用性
  isAvailable(): Promise<boolean>;
}

interface AIAgentSession {
  send(prompt: string): AsyncIterable<AIAgentEvent>;
  close(): void;
}

interface RunOptions {
  cwd?: string;
  systemPrompt?: string;
  allowedTools?: string[];
  disallowedTools?: string[];
  maxBudgetUsd?: number;
  model?: string;
  timeoutMs?: number;
  outputSchema?: object;
}

interface SessionOptions {
  cwd?: string;
  systemPrompt?: string;
  model?: string;
}
```

### Adapter 实现

| Adapter | `run()` 底层 | `createSession()` 底层 |
|---------|-------------|----------------------|
| `ClaudeCodeAdapter` | `claude -p --output-format stream-json` | `claude mcp serve` |
| `CursorAgentAdapter` | `agent -p --output-format stream-json` | `agent --resume`（会话恢复） |

### 引擎配置（engines.yml）

```yaml
default: claude-code
fallback: cursor-agent
claude-code:
  model: claude-sonnet-4-20250514
  maxBudgetUsd: 5
  allowedTools: [Read, Glob, Grep]
cursor-agent:
  model: gpt-5
  force: true
```

### 关键设计决策

- 所有 adapter 实现同一接口，上层代码只依赖 `AIAgentAdapter`
- `run()` 返回 `AsyncIterable`，支持流式处理（进度条、实时更新）
- 评估场景只开放只读工具（Read、Glob、Grep），禁止文件写入
- 超时和预算双重兜底，防止失控

## 五、文档解析层

### 统一接口

```typescript
interface ParserAdapter {
  readonly supportedExtensions: string[];
  parse(filePath: string): Promise<ParsedDocument>;
}

interface ParsedDocument {
  title?: string;
  author?: string;
  text: string;       // 纯文本全文
  metadata?: Record<string, any>;
}
```

### 各格式实现

| 格式 | 库 | 说明 |
|------|-----|------|
| TXT | `chardet` + `iconv-lite` | 自动检测编码（GBK/UTF-8等），转为 UTF-8 |
| EPUB | `epub` / `epub2md` | 提取章节 HTML → 剥离标签 → 纯文本 |
| PDF | `pdf-parse` | 基于 pdf.js，纯 JS 无原生依赖 |
| DOCX | `mammoth` | DOCX → 纯文本/Markdown |

### 自动注册

`registry.ts` 根据文件扩展名自动选择解析器，新增格式只需实现 `ParserAdapter` 接口并注册。

## 六、Web Dashboard 与 HTML 报告

### 1. 交互式 HTML 报告（独立文件）

- 每次评估生成一个自包含的 HTML 文件（内嵌 ECharts、CSS、数据）
- 浏览器直接打开，无需服务器

**报告页面结构：**
```
┌─────────────────────────────────────────┐
│  📊 Novel Eval 报告                      │
│  书名 · 作者 · 总字数 · 章节数 · 评分    │
├─────────────┬───────────────────────────┤
│  综合评分     │  五维雷达图              │
│  (大字居中)   │  (ECharts, 可悬停看子项) │
├─────────────┴───────────────────────────┤
│  五维详细分析（可折叠卡片）               │
│  ├── 故事架构：评分 + 分析 + 原文引用      │
│  ├── 人物塑造：评分 + 分析 + 原文引用      │
│  ├── 文笔质量：评分 + 分析 + 原文引用      │
│  ├── 情感共鸣：评分 + 分析 + 原文引用      │
│  └── 市场潜力：评分 + 分析 + 原文引用      │
├─────────────────────────────────────────┤
│  🕸 人物关系图谱（力导向图，可拖拽）       │
│  节点=角色, 边=关系, 颜色=阵营, 大小=重要度│
│  点击节点 → 跳转该角色首次出场/关键章节     │
├─────────────────────────────────────────┤
│  📈 情绪节奏曲线（面积图）               │
│  X轴=章节, Y轴=张力值(0-100)            │
│  点击数据点 → 跳转到该章节详情            │
├─────────────────────────────────────────┤
│  📊 章节评分趋势（折线图）               │
│  五维分色折线 + 综合评分线               │
│  点击 → 展开该章节评分卡片 + 原文引用      │
├─────────────────────────────────────────┤
│  💡 改进建议                             │
│  按维度分组，每条建议附原文引用           │
│  点击引用 → 展开上下文段落                │
└─────────────────────────────────────────┘
```

**Drill-down 交互：**
- 雷达图悬停 → 显示子项评分
- 人物关系节点点击 → 角色详情 + 首次出场章节 + 关键章节列表
- 情绪曲线数据点点击 → 跳转到章节评分卡片
- 章节趋势线点击 → 展开该章五维评分 + 分析 + 原文引用
- 建议中的引用点击 → 展开原文上下文段落

### 2. 本地 Web Dashboard（管理界面）

Node 后端常驻服务，功能：
- 上传/管理小说文件
- 发起评估任务，实时进度显示（解析中/分章中/第3/50章评估中/全局分析中）
- 历史评估列表，可回看旧报告
- 评估配置（调整权重、切换模型、并发数）
- Claude Code / Cursor Agent 状态监控

**技术选型：**
- 图表：ECharts（力导向图支持好，中文友好）
- HTML 报告：ECharts CDN 内嵌，零依赖单文件
- Web Dashboard 前端框架：待定

## 八、关键类型

```typescript
type TaskStatus =
  | 'pending' | 'parsing' | 'splitting'
  | 'mapping' | 'reducing' | 'generating'
  | 'completed' | 'failed'

interface EvaluationTask {
  id: string
  filePath: string
  fileName: string
  format: 'txt' | 'epub' | 'pdf' | 'docx'
  status: TaskStatus
  progress: { current: number; total: number; message: string }

  completedAt?: Date
  resultFile?: string
  reportFile?: string
}
```
